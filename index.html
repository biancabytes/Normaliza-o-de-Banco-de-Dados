Normalização de banco de dados

Bianca Vitoria e Beatriz Félix

Introduão

O que é a normalização de banco de dados e para o que ela serve?

Normalização é o processo de organização de dados em um banco de dados. Isso inclui a criação de tabelas e o estabelecimento de relações entre essas tabelas de acordo com as regras projetadas para proteger os dados e tornar o banco de dados mais flexível, eliminando a redundância e a dependência inconsistente.

Há algumas regras para normalização do banco de dados. Cada regra é chamada de "forma normal". Se a primeira regra for observada, diz-se que o banco de dados está na "primeira forma normal ". Se as três primeiras regras forem observadas, o banco de dados será considerado na "terceira forma normal". Embora outros níveis de normalização sejam possíveis, a terceira forma normal é considerada o nível mais alto necessário para a maioria dos aplicativos.

Uma forma normal é uma regra de deve ser aplicada na construção das tabelas do banco de dados para que estas fiquem bem projetadas. Segundo autores, existem 4 formas normais. Neste trabalho iremos falar sobre as 3 primeiras, sendo as principais:
- 1ª forma normal
- 2ª forma norma
- 3ª forma normal


Desenvolvimento

Formas Normais

Com o banco de dados construído, devem-se aplicar as 3 formas normais em cada tabela, ou grupo de tabelas relacionadas. As formas têm uma ordem e são dependentes, isto é, para se aplicar a segunda norma, deve-se obrigatoriamente ter aplicado a primeira e assim por diante.


1ª Forma normal: Verificação de Tabelas Aninhadas.

O que é: A 1ª forma normal estabelece que cada célula de uma tabela deve conter apenas um valor atômico e indindivisível. Ou seja, não podem existir grupos repetidos de valores em uma única célula.
Para uma tabela estar na primeira forma normal ela não deve conter tabelas aninhadas.

Um jeito fácil de verificar esta norma é fazer uma leitura dos campos das tabelas fazendo a pergunta: Este campo depende de qual?

Exemplo 1: Separar colunas com múltiplos valores em colunas únicas para cada valor.
Exemplo 2: Garantir que não existam grupos de repetição na tabela.


2ª Forma Normal: Verificação de Dependências Parciais

O que é: A 2ª forma normal exige que todas as colunas não-chave dependam completamente da chave primária da tabela. Ou seja, não pode haver dependência parcial.
Para uma tabela estar na segunda formal, além de estar na primeira forma ela não deve conter dependências parciais.

Um jeito de verificar esta norma é refazer a leitura dos campos fazendo a pergunta: Este campo depende de toda a chave? Se não, temos uma dependência parcial.

Exemplo 1: Criar tabelas separadas para remover dependências parciais.
Exemplo 2: Evitar que um atributo dependa de apenas parte da chave primária em tabelas com chave composta.


3ª Forma Normal: Verificação de Dependências Transitivas

O que é: A 3ª forma normal exige que todas as colunas não-chave dependam diretamente da chave primária e não de outras colunas não-chave. Ou seja, não pode haver dependência transitiva.
Para uma tabela estar na segunda formal, além de estar na segunda forma ela não deve conter dependências transitivas.

Um jeito de verificar esta norma é refazer a leitura dos campos fazendo a pergunta: Este campo depende de outro que não seja a chave? Se Sim, temos uma dependência transitiva.

Exemplo 1: Separar dados derivados para evitar duplicidade.
Exemplo 2: Eliminar colunas que dependam de outras colunas não-chave.


Conclusão

Conclusão Pessoal

A normalização é crucial para manter a eficiência e a integridade de um banco de dados. Com ela, é possível evitar redundâncias e melhorar o desempenho de consultas, facilitando o gerenciamento dos dados.
